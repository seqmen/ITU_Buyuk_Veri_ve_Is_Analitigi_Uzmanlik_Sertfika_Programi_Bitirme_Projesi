Nüks Etme Tahminleme Projesi¶

Veri seti zmir Katip elebi niversitesi Atatrk Eitim ve Aratrma Hastanesi Kadn Hastalklar ve Doum Servisi'nden alnmtr. Veriler yayn aamasnda olan bir almaya ait olduu iin genel kullanma ak deildir.

Veri Seti Özellikleri¶

Onkoloji hastalarnn deerlerine gre oluturulmu olan veri setinde aadaki bilgiler kayt altna alnmtr:

  • Age : Hastann ya - Numerik
  • Parity : Hastann ka doum yapt - Numerik
  • Aaconcentration : Asetik Asit Konsantrasyounu - Kategorik, sadece %4 ve %8 olarak mevcut
  • hpv1618vsnon : HPV enfeksiyonu pozitif kma veya kmamas, varsa hangi tip(lerin) olduu - Kategorik
  • multihpv : Birden ok HPV enfeksiyonu geirmesi - Kategorik
  • firstcvs : lk alnan Smear sonucu - Kategorik
  • colposcopy : Smear sonucuna rahim azndan alnan biyopsi ilemi - Kategorik
  • firstcvsaftercolposcopy : Colposcopy sonras kontrol iin alnan Smear sonucu - Kategorik
  • ConisationTAH : Colposcopy sonucuna gre konizasyon(rahim aznn bir glmnn koni eklinde kartlmas) yaplmas ve Total Abdominal Histerektomi(rahim alnma operasyonu) - Kategorik
  • Conisationonly : Sadece konizasyon yaplmas - Kategorik
  • EGCI : Rahim az salg bezlerine yaylm(Epitelial Glandular Cervical Invasion) - Kategorik
  • Surgmargpositivity : Rahim az operasyonunda cerrahi snrlarda pozitif grnm - Kategorik
  • TAH : Total Abdominal Histerektomi(rahim alnma operasyonu) - Kategorik
  • cvsafterconisation : Konizasyon sonras Smear sonucu - Kategorik
  • ColpConTahUyumu : Colposcopy, Konizasyon ve TAH sonularnn uyumlar
  • anteriorposterior : n-arka yaylm derinlii - Numerik
  • transverse : Yanlara yaylm derinlii - Numerik
  • deep : Dikine yaylm derinlii - Numerik
  • pieces : Para says - Numerik
  • nks : Hastada kanserin tekrarlanmas - Kategorik

Kategorik verilerin kodlanması¶

Veri setimizde birok kategorik bilgi bulunmakta. Daha nce yaptmz listeye bakrarak bu kategorik verileri numerik deerlere atayp ilemlerimize devam edeceiz.

Aadaki kod paras listelenen kategorik veri stunlar ierisinde dnerek hepsindeki deerlere numerik labellar atamaktadr. Bu numerik labellarn kaybolmamas iin bu verileri de Google Sheet'te ayr ayr tablolara yazacaz. Bu sayede ileride yeni veriler geldiinde bu eitilmi modelimizi kullanmamz iin gerekli haritalamay yapabiliriz.

import xlsxwriter
import math
from xlsxwriter.utility import xl_range

labelsMatrix = list()


from sklearn.preprocessing import LabelEncoder
encoder = LabelEncoder()

categoryFieldsList = ['Aaconcentration','hpv1618vsnon','multihpv','firstcvs','colposcopy','firstcvsaftercolposcopy','ConisationTAH','Conisationonly','EGCI','Surgmargpositivity','TAH','cvsafterconisation','ColpConTahUyumu','pieces','nks']
for item in categoryFieldsList:
    df[item] = encoder.fit_transform(df[item])
    print(f"Kolon ad {item} olan deiken iin")
    label_value_matrix = []
    for id_, class_ in enumerate(encoder.classes_):
        if isinstance(class_, float) and math.isnan(class_):
            class_ = "NONE"
        print(f"'{class_}' snfna verilen numerik deer '{id_}' olarak belirlenmitir")
        label_value_matrix.append([class_,id_])
    labelsMatrix.append(pd.DataFrame(label_value_matrix, columns=['label','value']))
    
writer = pd.ExcelWriter('ITU_Project_Mappings.xlsx', engine='xlsxwriter')
print(len(categoryFieldsList))
for i in range(0,len(categoryFieldsList)-1):
    workbook = writer.book
    labelsMatrix[i].to_excel(
        writer, 
        categoryFieldsList[i],
        startcol=0,
        startrow=0
    )
writer.close()

Outlier Kontrolü¶

Stats Model - Backward Elimination¶

Modelimizi son haline getirirken "nks" deerine etki etmeyen veya etkisi az olan parametrelerin kartlmas gerekmektedir. Bunun iin iteratif olarak backward elimination methodunu kullanacaz. ncelikle btn parametrelerin olduu bir model oluturup, bu modelden sras ile "p" deeri "0,05"in stnde olanlar kartacaz. "p" deeri "0,05"in stnde herhangi bir deer kalmadnda bu bizim renme modellerimizi iin kullanacamz veri seti olacaktr.

def is_there_exceeding_p(dataframe):
    df_transpose = dataframe.T
    for item in df_transpose:
        p_value = df_transpose[item].iloc[3]
        if p_value > 0.05 :
            return item
    return "none"

while 1:
    lm = sm.OLS(y,x)
    model = lm.fit()
    model_summary = model.summary()
    model_typed = model_summary.tables[1].as_html()
    model_DF = pd.read_html(model_typed, header=0, index_col=0)[0].sort_values(by=['P>|t|'],ascending=False)
    model_DF
    value = is_there_exceeding_p(model_DF)
    if value != "none" :
        columnList = list()
        columnList.append(value)
        print("'" + value +"', p-deeri ""0.05"" stnde olduu iin veri setinden kartlmtr.")
        x.drop(columns=columnList,inplace=True)
    else :
        break

Tekrar Ölçütleme ve Standartlaştırma¶

Baz algoritmalar ltlemere gre farkl performans gstermektedir.

Bunlardan biri k-nearest-neighbor algoritmasdr. Bizde bu ekilde bir durum olmasa bile buna ynelik almay aadaki gibi min-max scaler ile yapabiliriz.

Bu ltlemede btn deerler her kolon iin 0 ile 1 arasnda olacak ekilde dzenlenir. Genel olarak forml u ekildedir:

$x'_i = \frac{x_i - min(x)}{max(x) - min(x)}$

$min(x)$, $x$ ierisindeki en dk deerli veriyi, $x'_i$, $x_i$'in 0-1 arasna ltlenmi halini gstermektedir.
